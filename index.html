Derivadas por diferencias finitas

Con el desarrollo en serie de Taylor para $f(x)$ en el entorno de $x_0$ \begin{equation} f(x_0 + h) = f(x_0) + h \left.\frac{d f(x)}{dx}\right|_{x_0} + \frac{h^2}{2!} \left.\frac{d^2f(x)}{dx^2}\right|_{x_0} + ... \label{eq:Taylor} \end{equation} y haciendo una discretización del espacio \[{x \to \{x_0, x_2, ... , x_N\}}\] \[x_{j+1} - x_j = h\,\,\,\,\, \forall j\] \[f_j \equiv f(x_j)\,\,\,\,\, \forall j,\] despejando se puede obtener una aproximación para la derivada de la función: \begin{equation} \left.\frac{d f(x)}{dx}\right|_{x_j} = \frac{f_{j+1} - f_j}{h} + o(h). \label{eq:oh1} \end{equation} Esto se puede implementar en C++ mediante la función: 

// Derivative o(h) not centered
template  <class Type>
Type d_h1(Type f(Type), Type x, Type h)
{
  return (f(x + h) - f(x)) / h;
}

En este caso se utiliza una función template para reutilizarla con datos float y double que será interesante de ver luego. La derivada obtenida en \eqref{eq:oh1} depende linealmente de $h$, por lo que para obtener la derivada con una mejor precisión es necesario hacer $h$ más pequeño.

Se pueden hacer cosas más inteligentes con \eqref{eq:Taylor}. Si, por ejemplo, utilizamos $f(x_0 \pm h)$ y sumamos convenientemente obtenemos \begin{equation} \left.\frac{d f(x)}{dx}\right|_{x_j} = \frac{f_{j+1} - f_{j-1}}{2h} + o(h^2), \label{eq:oh2} \end{equation} con lo cual, por virtualmente el mismo esfuerzo (dos evaluaciones de $f$ y las mismas operaciones algebráicas) que \eqref{eq:oh1} obtenemos una precisión un orden de magnitud mejor.

Si se quiere se puede seguir jugando con \eqref{eq:Taylor} y obtener cosas cada vez más precisas, como \begin{equation} \left.\frac{d f(x)}{dx}\right|_{x_j} = \frac{-f_{j+2} + 8f_{j+1} - 8f_{j-1} + f_{j-2}}{12h} + o(h^4). \label{eq:oh4} \end{equation} En este último caso ya el cálculo es un poco más costoso, necesita 4 evaluaciones de $f$, pero la precisión aumentó en dos órdenes. 

// Derivative o(h^2) centered
template <class Type>
Type d_h2(Type f(Type), Type x, Type h)
{
  return (f(x + h) - f(x - h)) * 0.5 / h;
}

// Derivative o(h^4) centered
template <class Type>
Type d_h4(Type f(Type), Type x, Type h)
{
  return (-f(x + 2 * h) + 8 * f(x + h) - 8 * f(x - h) + f(x - 2 * h)) / (12 * h);
}

Para poner a prueba estas fórmulas calcularemos la derivada de $\sin(x)$ en $\pi/6$ utilizando las fórmulas de diferencias finitas y comparando con el valor exacto de la derivada $\cos(x)$ a través del error \[ \varepsilon = \left| \left.\frac{d\sin(x)}{dx}\right|_{x_0} - \cos(x_0) \right|. \] 

// Error calculation from method and actual derivative
template <class Type>
Type err(Type method(Type(Type), Type, Type), Type f(Type), Type df(Type), Type x, Type h)
{
  return abs(method(f, x, h) - df(x));
}

// Templated function
template <class Type>
Type f(Type x)
{
  return sin(x);
}

// Templated derivative
template <class Type>
Type df(Type x)
{
  return cos(x);
}

Error del cálculo de la derivada de $\sin(x)$ en $\pi/6$ utilizando las fórmulas \eqref{eq:oh1}, \eqref{eq:oh2} y \eqref{eq:oh4}. El error de aproximación supera al de truncamiento para $h$ pequeño.

Como era de esperar, la precisión es mejor cuanto mayor es el orden al que se trunca la serie \eqref{eq:Taylor} y el error disminuye a medida que $h$ se hace menor. Si hiciéramos en cálculo a lápiz bastaría el primer método tomando un $h$ tan pequeño como quisiéramos para asegurar la precisión buscada. Pero la representación de punto flotante de las computadoras para los números reales tiene tiene asociados errores de aproximación, por lo que a partir de cierto valor de $h$ se sobrepone al error de truncamiento. ¡He ahí el valor de las fórmulas de órdenes superiores!
